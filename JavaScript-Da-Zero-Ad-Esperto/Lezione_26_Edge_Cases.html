*6\. Casi Speciali e "Edge Cases"*

Esistono delle eccezioni fondamentali alle regole di conversione, in particolare quando si utilizza l'operatore di uguaglianza (==).

*1\. La Particolarità di null e undefined*

Sebbene matematicamente null venga convertito in 0 (come visto nel Caso 4: 2 + null = 2), questa conversione *NON avviene durante un controllo di uguaglianza*24.

*Regola:* In un'uguaglianza debole, null è uguale solo a undefined o a se stesso. Non è uguale a nient'altro (nemmeno a 0 o false)25.

JavaScript

// Aritmetica (Coercizione attiva)

2 + null; // 2 (null diventa 0\)

// Uguaglianza (Eccezione\!)

null == 0; // false

// Qui null NON viene trasformato in 0, quindi il confronto fallisce.

// Relazioni di uguaglianza valide

null == null; // true

null == undefined; // true

*2\. Il Paradosso di NaN*

NaN (Not a Number) è un valore speciale che indica il fallimento di un'operazione numerica.

*Regola: NaN non è mai uguale a nulla, nemmeno a se stesso*26.

JavaScript

const valoreErrato = NaN;

// Uguaglianza fallisce sempre

NaN == NaN; // false \[cite: 20\]

Per verificare se un valore è NaN, non si può usare l'operatore ==, ma è necessario utilizzare la funzione specifica isNaN().
