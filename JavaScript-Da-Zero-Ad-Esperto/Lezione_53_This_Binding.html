this: Binding Rules

Lezione 53 - Corso JavaScript

*Regole di Binding per this*

JavaScript ha quattro regole principali per determinare il valore di this:

*1\. Default Binding (Binding Predefinito)*

Quando una funzione viene invocata senza un contesto specifico:

function fn() {

console.log(this);

}

fn();

// Output: Window (in modalità non-strict)

// Output: undefined (in modalità strict)

*2\. Implicit Binding (Binding Implicito)*

Quando una funzione viene chiamata come metodo di un oggetto:

const obj = {

name: "Oggetto",

fn: function() {

console.log(this.name);

}

};

obj.fn();

// Output: "Oggetto"

*3\. Explicit Binding (Binding Esplicito)*

Utilizzando call(), apply() o bind():

function greet() {

console.log(\`Ciao, sono ${this.name}\`);

}

const person = { name: "Mario" };

greet.call(person);

// Output: "Ciao, sono Mario"

greet.apply(person);

// Output: "Ciao, sono Mario"

const boundGreet = greet.bind(person);

boundGreet();

// Output: "Ciao, sono Mario"

*4\. new Binding*

Quando una funzione viene usata come costruttore con new:

function Persona(nome) {

this.nome = nome;

}

const p1 = new Persona("Luigi");

console.log(p1.nome);

// Output: "Luigi"

*Arrow Functions e this*

Le *arrow functions* hanno un comportamento diverso rispetto alle funzioni tradizionali:

  • Non hanno il proprio this
  • Ereditano this dal contesto lessicale circostante
  • Il valore di this è determinato al momento della definizione, non dell'invocazione

const obj = {

name: "Oggetto",

regularFn: function() {

console.log(this.name);

},

arrowFn: () => {

console.log(this.name);

}

};

obj.regularFn();

// Output: "Oggetto"

obj.arrowFn();

// Output: undefined (this è Window nel contesto globale)

*Perdita del Contesto (Context Loss)*

Un problema comune è la perdita del contesto quando si passa un metodo come callback:

*Problema:*

const obj = {

name: "Oggetto",

greet: function() {

console.log(\`Ciao da ${this.name}\`);

}

};

setTimeout(obj.greet, 1000);

// Output: "Ciao da undefined" (this è Window)

*Soluzione 1: Arrow Function*

setTimeout(() => obj.greet(), 1000);

// Output: "Ciao da Oggetto"

*Soluzione 2: bind()*

setTimeout(obj.greet.bind(obj), 1000);

// Output: "Ciao da Oggetto"

*Strict Mode e this*

In *strict mode*, il comportamento di this cambia:

"use strict";

function test() {

console.log(this);

}

test();

// Output: undefined (non Window\!)

Questo aiuta a individuare errori in cui this potrebbe non essere quello che ci aspettiamo.
