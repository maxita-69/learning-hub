*Lezione: Operatori Aritmetici e Numeri*

*1\. Introduzione*

La lezione precedente probabilmente non è stata leggera. In questa lezione ci esercitiamo un po' con i numeri.

*2\. Operatori Aritmetici Fondamentali*

Per i nostri calcoli aritmetici abbiamo gli operatori aritmetici:

  • + per la somma
  • - per la sottrazione
  • / per la divisione
  • * per la moltiplicazione

Esempio:

let risultato = 10 + 3 * 2;

console.log(risultato); // 16

Abbiamo correttamente 16 perché alla moltiplicazione viene data precedenza.

*3\. Uso delle Parentesi*

Se volessimo prima sommare 10 + 3, poi moltiplicare per due, lo possiamo fare con le parentesi tonde:

let risultato = (10 + 3) * 2;

console.log(risultato); // 26

In questo caso verrà effettuato prima la somma 10 + 3 e poi la moltiplicazione per due. Ed infatti abbiamo 26 esattamente.

*4\. Operatore Modulo*

L'operatore modulo restituisce il resto della divisione tra due numeri.

Esempi:

console.log(10 % 3); // 1  
// Perché 10 diviso 3 = 3 col resto di 1

console.log(10 % 4); // 2    
// Perché 10 diviso 4 = 2 con il resto di 2

console.log(10 % 5); // 0

// Perché 10 diviso 5 = 2 col resto di zero

*5\. Operatore di Elevazione a Potenza*

Possiamo elevare un numero ad un esponente con il doppio asterisco:

console.log(2 * 3); // 8*

// Due elevato alla terza = 8

*6\. Operatori di Incremento e Decremento*

Post-incremento:

let num = 0;  
console.log(num++); // 0

console.log(num);   // 1

Con il post-incremento prima restituisce il numero e poi lo incrementa.

Pre-incremento:

javascript

let num = 0;

console.log(++num); // 1

Con il pre-incremento prima viene incrementato il valore e poi restituito.

Post-decremento:

javascript

let num = 5;  
console.log(num--); // 5

console.log(num);   // 4

Pre-decremento:

javascript

let num = 5;

console.log(--num); // 4

*7\. Oggetto Number vs Tipo Primitivo*

In JavaScript si dice che gli oggetti sono "first-class". Quindi non dobbiamo meravigliarci che esiste non solo il tipo primitivo number, ma anche l'oggetto Number.

Numero come primitivo:

let num = 5;

Numero come oggetto:

javascript

let numeroOggetto = new Number(110);

Non è argomento di questo capitolo, quindi non preoccupiamoci della sintassi.

*8\. Range dei Numeri Sicuri*

Al termine della precedente lezione abbiamo detto che esiste un range composto da un numero minimo e un numero massimo che può essere rappresentato in JavaScript in modo sicuro.

console.log(Number.MAX_SAFE_INTEGER); // Limite massimo

console.log(Number.MIN_SAFE_INTEGER); // Limite minimo

Verifica sicurezza numeri:

javascript

let numeroGrande = Number.MAX_SAFE_INTEGER + 10;

console.log(Number.isSafeInteger(numeroGrande)); // false

Se il numero supera il limite di MAX_SAFE_INTEGER, avremo false.

*9\. L'oggetto Math*

Come già accennato nel primo capitolo, abbiamo anche l'oggetto Math che fornisce molte proprietà e metodi per lavorare con i numeri.

Numero casuale tra 0 e 1:

console.log(Math.random());

Numero casuale tra 0 e 1000:

javascript

let casuale = Math.random() * 1000;

console.log(casuale); // Es: 590.086...

Numero intero casuale tra 0 e 1000:

javascript

let casuale = Math.trunc(Math.random() * 1000);

console.log(casuale); // Es: 665, 402, ecc.

Continuiamo ad aggiornare e vediamo che ogni volta il valore cambia.

Perfetto per questa lezione è tutto, ci vediamo nella prossima lezione.

*Lezione: Operatori Aritmetici, Numeri e Approfondimenti in JavaScript1. Introduzione: Un Passo Indietro e uno Avanti*

Se la lezione precedente ha richiesto una concentrazione elevata, questa sessione è pensata per consolidare le basi e, al contempo, introdurre concetti più sfumati riguardanti l'uso dei numeri e degli operatori aritmetici in JavaScript. L'obiettivo è esercitarsi con la manipolazione numerica, che è fondamentale in quasi ogni aspetto della programmazione.*2\. Gli Operatori Aritmetici Fondamentali: L'ABC del Calcolo*

Gli operatori aritmetici sono i mattoni di qualsiasi espressione matematica. In JavaScript, sono rappresentati come segue:

  • *`+` (Somma):* Per addizionare due operandi.
  • *`-` (Sottrazione):* Per sottrarre il secondo operando dal primo.
  • *`/` (Divisione):* Per dividere il primo operando per il secondo, restituendo un risultato decimale se necessario.
  • *`` (Moltiplicazione):** Per moltiplicare due operandi.

È cruciale ricordare la *precedenza degli operatori*. In assenza di parentesi, JavaScript (come la matematica standard) valuta prima la moltiplicazione e la divisione, e poi l'addizione e la sottrazione.

*Esempio di Precedenza:*

let risultato = 10 + 3 * 2;

// La moltiplicazione (3 * 2 = 6\) ha la precedenza.

// L'espressione diventa 10 + 6\.

console.log(risultato); // 16

*3\. L'Uso Strategico delle Parentesi Tonche*

Quando si desidera alterare l'ordine naturale di precedenza, le *parentesi tonde `()`* sono lo strumento essenziale. Le operazioni racchiuse tra parentesi vengono sempre eseguite per prime.

*Esempio di Parentesi:*

let risultato = (10 + 3\) * 2;

// La somma (10 + 3 = 13\) viene eseguita per prima grazie alle parentesi.

// L'espressione diventa 13 * 2\.

console.log(risultato); // 26

*4\. L'Operatore Modulo (`%`): Il Resto della Divisione*

L'operatore modulo (`%`) è spesso sottovalutato, ma è estremamente utile in scenari come la verifica di numeri pari/dispari, la ciclicità o la distribuzione uniforme. Non restituisce il quoziente, ma il *resto intero* della divisione tra due numeri.

*Esempi di Modulo:*

console.log(10 % 3); // 1 (Perché 10 / 3 = 3 con resto 1\)

console.log(10 % 4); // 2 (Perché 10 / 4 = 2 con resto 2\)

console.log(10 % 5); // 0 (Perché 10 / 5 = 2 con resto 0 - utile per verificare i multipli\!)

*5\. L'Operatore di Elevazione a Potenza (``): Esponenti in JavaScript*

Per elevare un numero a una determinata potenza (esponente), JavaScript offre l'operatore *doppio asterisco ``*.

*Esempio di Potenza:*

console.log(2 * 3); // 8 (Due elevato alla terza = 2 2 * 2\)

console.log(5 ** 0.5); // 2.23606797749979 (Radice quadrata di 5\)

(Alternativa: Per operazioni matematiche più complesse, si può usare anche `Math.pow(base, esponente)`).*6\. Operatori di Incremento e Decremento: Semplificare le Variazioni di Unità*

Questi operatori sono scorciatoie per aggiungere o sottrarre 1 a una variabile e sono fondamentali nei cicli e nelle iterazioni. È cruciale distinguere tra la versione post- e pre-.*Post-incremento (`num++`) e Post-decremento (`num--`)*

Il valore della variabile viene *prima restituito/utilizzato nell'espressione e poi incrementato/decrementato*.

let numPost = 0;

console.log(numPost++); // 0 (Stampa il valore corrente, poi lo incrementa a 1\)

console.log(numPost); // 1

*Pre-incremento (`++num`) e Pre-decremento (`--num`)*

Il valore della variabile viene *prima incrementato/decrementato e poi restituito/utilizzato* nell'espressione.

let numPre = 0;

console.log(++numPre); // 1 (Incrementa il valore a 1, poi stampa il nuovo valore)

let numPreDec = 5;

console.log(--numPreDec); // 4 (Decrementa il valore a 4, poi stampa il nuovo valore)

*7\. Il Dualismo dei Numeri: Tipo Primitivo vs. Oggetto Number*

In JavaScript, i tipi di dati si dividono in primitivi (come `number`, `string`, `boolean`, ecc.) e oggetti. Il tipo `number` (primitivo) è il più comune per le operazioni numeriche, ma esiste anche l'oggetto `Number`.

  • *Numero Primitivo:* È un valore semplice e diretto.

let numPrimitivo = 5;

  • *Oggetto `Number`:* È un'istanza dell'oggetto `Number`, e sebbene JavaScript lo gestisca in modo simile al primitivo in molti contesti, è concettualmente diverso e fornisce accesso a proprietà e metodi specifici.

let numeroOggetto = new Number(110);

(Approfondiremo la differenza tra primitivi e oggetti, in particolare i "wrapper" di oggetti, in capitoli successivi. Per ora, è sufficiente sapere che la maggior parte delle volte lavoreremo con il tipo primitivo `number`).*8\. Il Range di Sicurezza dei Numeri in JavaScript*

JavaScript utilizza la rappresentazione a 64 bit in virgola mobile (IEEE 754), il che implica che non tutti gli interi possono essere rappresentati con precisione. Esiste un intervallo di "numeri interi sicuri": quelli che possono essere rappresentati in modo univoco e preciso.

  • *`Number.MAX_SAFE_INTEGER`:* Rappresenta il numero intero massimo sicuro (2^53 - 1).
  • *`Number.MIN_SAFE_INTEGER`:* Rappresenta il numero intero minimo sicuro (-(2^53 - 1)).

*Verifica della Sicurezza:*

console.log(Number.MAX_SAFE_INTEGER);

console.log(Number.MIN_SAFE_INTEGER);

let numeroGrande = Number.MAX_SAFE_INTEGER + 10;

// Oltre questo limite, le operazioni aritmetiche possono non essere accurate.

console.log(Number.isSafeInteger(numeroGrande)); // false

(Per numeri interi estremamente grandi che superano questo limite, si ricorre al tipo `BigInt`, introdotto più recentemente in JavaScript, ma che non è oggetto di questa lezione).*9\. L'Oggetto `Math`: Funzioni Matematiche Avanzate*

L'oggetto `Math` è un oggetto built-in che non ha un costruttore e fornisce proprietà e metodi per costanti matematiche (come `Math.PI`) e funzioni. È la nostra libreria standard per operazioni matematiche più complesse.*Generazione di Numeri Casuali (`Math.random`)*

Il metodo `Math.random()` restituisce un numero in virgola mobile pseudo-casuale, compreso tra 0 (incluso) e 1 (escluso).

// Numero casuale tra 0 e 1

console.log(Math.random());

// Numero casuale tra 0 e 1000 (escluso)

let casualeFloat = Math.random() * 1000;

console.log(casualeFloat); // Es: 590.086...

// Numero intero casuale tra 0 e 999 (intervallo di 1000 numeri)

// Usiamo Math.trunc() per eliminare la parte decimale.

let casualeIntero = Math.trunc(Math.random() * 1000);

console.log(casualeIntero); // Es: 665, 402, ecc.

(Altri metodi utili di `Math` includono `Math.floor()` (arrotonda per difetto), `Math.ceil()` (arrotonda per eccesso), `Math.round()` (arrotonda all'intero più vicino) e `Math.sqrt()` (radice quadrata)).*10\. Conclusione: Padronanza del Calcolo*

Avere una solida comprensione degli operatori aritmetici, della loro precedenza e degli strumenti come l'oggetto `Math` e la consapevolezza del safe integer range è fondamentale per scrivere codice JavaScript robusto e affidabile.
