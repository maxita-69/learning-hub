Funzioni Pure

Lezione 40 - Corso JavaScript

*1\. Definizione di Funzione Pura*

Nel paradigma funzionale, una funzione è considerata *pura* quando rispetta due regole fondamentali:

1. *Determinismo: Il risultato della funzione dipende esclusivamente dai dati passati in input (argomenti). A fronte dello stesso identico input, la funzione deve restituire sempre* lo stesso identico output.

2. *Assenza di Effetti Collaterali (Side Effects):* La funzione non deve modificare lo stato esterno, né produrre cambiamenti osservabili al di fuori del suo scopo (come modificare variabili globali, alterare il DOM o stampare in console).

*2\. Funzioni Impure e Non Deterministiche*

Per comprendere meglio, partiamo da un contro-esempio. Analizziamo una funzione che calcola una somma ma dipende da una variabile esterna.

*Esempio di Codice Impuro*

Definiamo una variabile somma nel contesto globale e una funzione calcola che agisce su di essa.

JavaScript

// Variabile definita nel contesto globale (esterno alla funzione)

let somma = 0;

function calcola(a, b) {

// La funzione dipende e modifica una variabile esterna

somma = a + b;

return somma;

}

// Invocazione

console.log(calcola(2, 3)); // Output: 5

console.log(somma); // Output: 5 (La variabile esterna è stata modificata)

*Perché questa funzione è impura?*

1. *Dipendenza Esterna:* Il suo funzionamento è legato alla variabile somma che si trova fuori dal suo scope.

2. *Effetto Collaterale:* Modifica la variabile somma (Side Effect). Se invocassimo la funzione più volte o se un'altra parte del codice modificasse somma, lo stato dell'applicazione diventerebbe difficile da tracciare.

*Nota sui Side Effects: Anche l'uso di console.log() all'interno di una funzione è tecnicamente considerato un effetto collaterale*, poiché modifica lo stato della console (un'entità esterna alla funzione) mostrando un'informazione a video.

*3\. Refactoring verso una Funzione Pura*

Trasformiamo ora la funzione precedente in una funzione pura. L'obiettivo è eliminare le dipendenze esterne e far sì che l'output sia determinato solo dagli input.

*Esempio di Codice Puro*

JavaScript

function calcola(a, b) {

// La funzione lavora solo con i parametri 'a' e 'b'

let risultato = a + b;

// Restituisce il valore senza modificare nulla all'esterno

return risultato;

}

// Invocazione

console.log(calcola(1, 2)); // Output: 3

console.log(calcola(1, 2)); // Output: 3 (Sempre lo stesso risultato per lo stesso input)

*Vantaggi dell'approccio Puro:*

  • *Prevedibilità:* Se passiamo 1, 2, 3, otterremo sempre 6, indipendentemente dallo stato del resto dell'applicazione.
  • *Manutenibilità e Debug:* È più facile capire cosa fa la funzione e trovare errori, poiché non ci sono variabili nascoste che ne influenzano il comportamento.

*4\. Riepilogo del Capitolo: Le Funzioni in JavaScript*

Con questa lezione concludiamo il capitolo dedicato alle funzioni. Ecco una sintesi dei concetti chiave trattati finora:

1. *Oggetti di Prima Classe (First-Class Citizens):* Le funzioni in JS sono trattate come qualsiasi altro valore (possono essere assegnate a variabili, passate come argomenti, restituite da altre funzioni).

2. *Callback:* Una funzione passata come parametro ad un'altra funzione.

3. *Higher-Order Functions (Funzioni di Ordine Superiore): Funzioni che ricevono altre funzioni come input o ne restituiscono una. Questo ci permette di lavorare a un livello di astrazione* più alto.

4. *Arrow Functions (ES6):* Una sintassi concisa per le funzioni anonime, spesso usata per le callback.

5. *Constructor Functions:* Funzioni utilizzate con l'operatore new per creare oggetti. Permettono di gestire l'ereditarietà tramite la proprietà prototype e l'uso di this per definire le proprietà dell'istanza.

6. *Pure vs Impure:* La distinzione fondamentale basata su determinismo ed effetti collaterali.

La comprensione profonda di questi meccanismi è la base per affrontare argomenti più complessi nello sviluppo di applicazioni JavaScript moderne.
