Object.defineProperty in JavaScript

Controllo Avanzato delle Proprietà degli Oggetti

Introduzione

Il metodo Object.defineProperty() permette di definire una nuova proprietà di un oggetto specificandone nel dettaglio tutte le caratteristiche. A differenza della normale assegnazione di proprietà (obj.prop = value), questo metodo offre un controllo granulare su come la proprietà si comporta.

Con Object.defineProperty() possiamo decidere:

  • Il valore della proprietà
  • Se la proprietà è enumerabile (visibile nei cicli)
  • Se la proprietà è scrivibile (modificabile)
  • Se la proprietà è configurabile (eliminabile/modificabile)

Sintassi di Base

Sintassi Completa

Object.defineProperty(oggetto, nomeProprietà, descrittore)
  • oggetto: L'oggetto su cui definire la proprietà
  • nomeProprietà: Il nome della nuova proprietà (stringa)
  • descrittore: Oggetto che descrive le caratteristiche della proprietà

Setup Iniziale

Iniziamo con un oggetto semplice contenente una sola proprietà:

// Oggetto iniziale con una proprietà
let obj = {
    p1: 10
};

Vogliamo ora aggiungere una seconda proprietà p2 utilizzando Object.defineProperty(), specificando esattamente come deve comportarsi.

I Descrittori di Proprietà

Il terzo parametro di Object.defineProperty() è un oggetto descrittore che può contenere diverse proprietà per configurare il comportamento della nuova proprietà.

Descrittore: value

Specifica il valore che la proprietà deve avere. 

// Definiamo le caratteristiche della nuova proprietà
let caratteristiche = {
    value: 20
};

Descrittore: enumerable

Determina se la proprietà è enumerabile, cioè se appare quando iteriamo l'oggetto con cicli come for...in o metodi come Object.keys(). 

let caratteristiche = {
    value: 20,
    enumerable: false // La proprietà non sarà visibile nei cicli
};
Valore di default: Se non specificato, enumerable è false per proprietà definite con Object.defineProperty(). Per proprietà create con assegnazione normale (obj.prop = value), invece, è true.

Esempio Completo: Enumerable

Definizione della Proprietà

// Oggetto iniziale
let obj = {
    p1: 10
};

// Caratteristiche della nuova proprietà
let caratteristiche = {
    value: 20,
    enumerable: false
};

// Definiamo la nuova proprietà p2
Object.defineProperty(obj, 'p2', caratteristiche);

Verifica: La Proprietà Esiste

Verifichiamo che la proprietà p2 sia stata aggiunta:

console.log(obj.p2);
Output: 20

La proprietà p2 esiste ed è accessibile, con il valore 20 come specificato.

Comportamento con for...in

Iteriamo l'oggetto con un ciclo for...in per vedere quali proprietà sono enumerabili: 

// Iteriamo sulle proprietà enumerabili
for (let prop in obj) {
    console.log(prop);
}
Output: p1

Come possiamo vedere, solo p1 viene visualizzata. La proprietà p2 non appare perché abbiamo impostato enumerable: false.

Rendere la Proprietà Enumerabile

Se impostiamo enumerable: true:

// Redefiniamo con enumerable: true
let caratteristiche = {
    value: 20,
    enumerable: true // Ora la proprietà sarà visibile
};

Object.defineProperty(obj, 'p2', caratteristiche);

// Iteriamo di nuovo
for (let prop in obj) {
    console.log(prop);
}
Output: p1 p2

Ora p2 viene elencata perché è diventata enumerabile.

Descrittore: writable

Il descrittore writable determina se il valore della proprietà può essere modificato dopo la creazione. Impostando writable: false, rendiamo la proprietà read-only (sola lettura).

writable: false

Quando una proprietà ha writable: false:

  • La proprietà non può essere modificata
  • Tentativi di assegnazione vengono ignorati silenziosamente (in modalità normale)
  • In strict mode, sollevano un TypeError

Esempio: Proprietà Non Scrivibile

// Oggetto iniziale
let obj = {
    p1: 10
};

// Caratteristiche con writable: false
let caratteristiche = {
    value: 20,
    enumerable: true,
    writable: false // Proprietà non modificabile
};

Object.defineProperty(obj, 'p2', caratteristiche);

// Verifichiamo il valore iniziale
console.log(obj.p2); // 20
Output: 20

Tentativo di Modifica

Proviamo ad assegnare un nuovo valore a p2:

// Tentativo di modificare la proprietà
obj.p2 = 1000;

// Verifichiamo se il valore è cambiato
console.log(obj.p2);
Output: 20

Il valore di p2 non è cambiato! Rimane 20 nonostante il tentativo di assegnazione. L'assegnazione obj.p2 = 1000 è stata semplicemente ignorata perché la proprietà ha writable: false.

Strict Mode e TypeError

In modalità strict, JavaScript adotta regole più rigorose e solleva errori per operazioni che in modalità normale vengono silenziosamente ignorate.

Abilitare Strict Mode

Per abilitare la modalità strict, aggiungiamo "use strict" all'inizio del nostro script: 

"use strict";

let obj = {
    p1: 10
};

let caratteristiche = {
    value: 20,
    enumerable: true,
    writable: false
};

Object.defineProperty(obj, 'p2', caratteristiche);

// Primo console.log
console.log(obj.p2); // 20

// Tentativo di modifica
obj.p2 = 1000; // â  Questo solleva un TypeError!

// Questo codice non verrà mai eseguito
console.log(obj.p2);
â TypeError: Cannot assign to read only property 'p2' of object '#<Object>'

In strict mode, il tentativo di modificare una proprietà con writable: false solleva un errore di tipo TypeError e l'esecuzione del programma JavaScript si interrompe.

⚠️ Importante: Il secondo console.log() non viene mai eseguito perché l'errore interrompe l'esecuzione del codice alla riga precedente.

Differenze tra Modalità Normale e Strict

Comportamento Modalità Normale Strict Mode
Modifica proprietà non scrivibile Ignorata silenziosamente Solleva TypeError
Esecuzione del codice Continua normalmente Si interrompe all'errore
Debugging Difficile individuare errori Errori espliciti e chiari
Sicurezza del codice Minore (errori nascosti) Maggiore (errori evidenti)

Altri Descrittori

Oltre a value, enumerable e writable, esiste un quarto descrittore importante:

Descrittore: configurable

configurable

Determina se la proprietà può essere:

  • Eliminata dall'oggetto con delete
  • Modificata nei suoi descrittori (eccetto writable che può passare da true a false)

Default: false (quando si usa Object.defineProperty)

Esempio: Proprietà Non Configurabile

let obj = { p1: 10 };

Object.defineProperty(obj, 'p2', {
    value: 20,
    enumerable: true,
    writable: true,
    configurable: false // Proprietà non configurabile
});

// Tentativo di eliminare la proprietà
delete obj.p2;

console.log(obj.p2); // 20 - La proprietà non è stata eliminata
In strict mode: Tentare di eliminare una proprietà con configurable: false solleva un TypeError.

Riepilogo dei Descrittori

Descrittore Cosa Controlla Default (defineProperty) Default (assegnazione normale)
value Il valore della proprietà undefined Il valore assegnato
writable Se il valore può essere modificato false true
enumerable Se appare nei cicli (for...in, Object.keys) false true
configurable Se può essere eliminata o riconfigurata false true
Nota importante: I valori di default sono molto diversi tra proprietà create con Object.defineProperty() (più restrittive, tutti false) e proprietà create con assegnazione normale (più permissive, tutti true).

Casi d'Uso Pratici

1. Costanti Vere (Read-Only)

const config = {};

Object.defineProperty(config, 'API_KEY', {
    value: 'abc123xyz',
    writable: false,
    configurable: false
});

// La chiave API non può essere modificata o eliminata
config.API_KEY = 'hacked'; // Ignorato o TypeError in strict mode
console.log(config.API_KEY); // 'abc123xyz'

2. Proprietà Nascoste (Non Enumerabili)

const user = {
    name: "Mario",
    email: "mario@example.com"
};

// Aggiungiamo un ID interno non visibile nei cicli
Object.defineProperty(user, '_id', {
    value: 12345,
    enumerable: false // Non apparirà in for...in o Object.keys()
});

console.log(Object.keys(user)); // ['name', 'email']
console.log(user._id); // 12345 - accessibile ma non enumerabile

3. Metadati degli Oggetti

const product = {
    name: "Laptop",
    price: 999
};

// Aggiungiamo timestamp di creazione (read-only e nascosto)
Object.defineProperty(product, 'createdAt', {
    value: new Date(),
    writable: false,
    enumerable: false,
    configurable: false
});

// Il timestamp non può essere modificato o eliminato

4. Getter e Setter (Proprietà Calcolate)

Object.defineProperty() supporta anche getter e setter, proprietà di accesso invece che proprietà di dati: 

const person = {
    firstName: "Mario",
    lastName: "Rossi"
};

Object.defineProperty(person, 'fullName', {
    get: function() {
        return `${this.firstName} ${this.lastName}`;
    },
    set: function(name) {
        const parts = name.split(' ');
        this.firstName = parts[0];
        this.lastName = parts[1];
    },
    enumerable: true
});

console.log(person.fullName); // "Mario Rossi"
person.fullName = "Luigi Verdi";
console.log(person.firstName); // "Luigi"
console.log(person.lastName);  // "Verdi"

Best Practices

1. Usa Sempre Strict Mode

Abilita "use strict" per individuare errori più facilmente durante lo sviluppo.

2. Documenta le Proprietà Speciali

Quando usi Object.defineProperty() per creare proprietà con comportamenti non standard, documenta chiaramente il perché.

3. Considera le Alternative

Per casi semplici, considera se alternative moderne come:

  • Object.freeze() - Rende l'intero oggetto immutabile
  • Object.seal() - Impedisce l'aggiunta/rimozione di proprietà
  • Object.preventExtensions() - Impedisce solo l'aggiunta di nuove proprietà

4. Attenzione con configurable: false

Una volta impostato configurable: false, non è più possibile tornare indietro. Usa con cautela.

Metodi Correlati

Object.defineProperties()

Definisce multiple proprietà in una volta sola:

const obj = {};

Object.defineProperties(obj, {
    'property1': {
        value: 10,
        writable: true
    },
    'property2': {
        value: 20,
        writable: false
    }
});

Object.getOwnPropertyDescriptor()

Ottiene il descrittore di una proprietà esistente:

const descriptor = Object.getOwnPropertyDescriptor(obj, 'p2');
console.log(descriptor);
// { value: 20, writable: false, enumerable: true, configurable: false }

Conclusioni

Object.defineProperty() è uno strumento potente che offre controllo granulare sulle proprietà degli oggetti. I punti chiave da ricordare:

  1. value: Imposta il valore della proprietà
  2. enumerable: Controlla se la proprietà appare nei cicli (false per default con defineProperty)
  3. writable: Determina se il valore può essere modificato (false = read-only)
  4. configurable: Controlla se la proprietà può essere eliminata o riconfigurata
  5. Strict mode: Solleva TypeError invece di ignorare silenziosamente le violazioni, facilitando il debugging
Quando usarlo: Usa Object.defineProperty() quando hai bisogno di controllo preciso sulle caratteristiche di una proprietà, come creare costanti vere, proprietà nascoste, o getter/setter personalizzati.
