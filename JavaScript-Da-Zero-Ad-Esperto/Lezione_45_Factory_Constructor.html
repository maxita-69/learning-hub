Factory vs Constructor

Lezione 45 - Corso JavaScript

*5\. Creazione degli oggetti*

*5.1 Oggetti letterali e duplicazione del codice*

const user1 = {

name: "Mario",

descrivimi: function () {

console.log("Sono Mario");

}

};

const user2 = {

name: "Luigi",

descrivimi: function () {

console.log("Sono Luigi");

}

};

const user3 = {

name: "Anna",

descrivimi: function () {

console.log("Sono Anna");

}

};

Questa soluzione *viola il principio OOP di non duplicazione del codice*.

*5.2 Factory Function*

Per risolvere il problema si può usare una *factory function*.

function user(nome) {

const obj = {

name: nome,

descrivimi: function () {

console.log("Sono " + this.name);

}

};

return obj;

}

const user4 = user("Marco");

console.log(user4); — otteniamo un oggetto, con le proprietà descrivimi e name

Caratteristiche:

  • non utilizza `new`
  • restituisce esplicitamente un oggetto
  • permette di creare più istanze senza duplicare codice

stiamo invocando la funzione “user”, assegnando il valore di ritorno alla variabile user4

*5.3 Constructor Function*

Un’alternativa più compatta è la *constructor function*.

function Utenti(nome) {

this.name = nome;

this.descrivimi = () => ‘Nome:${this.name}’;

}

}

let user5=Utenti(“Noemi”)

Se invocata senza `new`:

const user5 = Utenti("Paolo");

console.log(user5); // undefined

Motivo:

  • la funzione non restituisce nulla
  • una funzione senza `return` restituisce `undefined`

Se invece usiamo `new`: (se mettiamo return 10, l’errore si elimina)

const user5 = new Utenti("Paolo");

console.log(user5);

JavaScript:

1. crea un nuovo oggetto

2. assegna `this` a quell’oggetto

3. restituisce automaticamente l’oggetto

l’oggetto che abbiamo adesso ha due proprietà

È ora possibile creare molte istanze:

const user6 = new Utenti("Giulia");

const user7 = new Utenti("Francesco");

*6\. Factory Function vs Constructor Function*

Entrambe permettono di:

  • creare più oggetti
  • evitare duplicazione del codice

Differenze principali:

  • la *constructor function* ha una sintassi più compatta
  • il comportamento di `this` e `new` è gestito internamente dal motore JavaScript
