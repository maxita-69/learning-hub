*1\. Modalità di Creazione degli Oggetti*

JavaScript offre diversi modi per creare oggetti. Sebbene la sintassi "letterale" sia la più comune, è importante conoscere l'alternativa basata sui costruttori nativi.

*Sintassi Letterale vs Costruttore*

JavaScript

// 1\. Sintassi Object Literal (Consigliata per oggetti singoli)

let obj = {};

// 2\. Sintassi con costruttore new Object

let obj2 = new Object();

La parola chiave new non è un'esclusiva degli oggetti generici, ma si applica anche ai tipi primitivi (creando i cosiddetti "Wrapper Objects"), anche se con comportamenti diversi:

JavaScript

let num = 10; // Numero primitivo (number)

let num2 = new Number(10); // Oggetto Number (object)

*2\. Le Constructor Functions*

Le *Constructor Functions sono normali funzioni che fungono da "stampo" (template) per creare oggetti. Per convenzione, il loro nome inizia sempre con la lettera maiuscola* (PascalCase) per distinguerle dalle funzioni ordinarie.

*Definizione Base*

Definiamo una funzione User vuota.

JavaScript

function User() {

// Corpo della funzione

}

console.log(User); // Visualizza il corpo della funzione

*Invocazione con new*

La differenza fondamentale risiede nel modo in cui la funzione viene invocata.

1. *Senza new:* Viene eseguita come una normale funzione. Poiché non ha un return, restituisce undefined.

2. JavaScript

let user1 = User();

console.log(user1); // Output: undefined

*3\. Gestione delle Proprietà: this vs prototype*

Quando dobbiamo creare molti oggetti simili (es. 1000 utenti), usare la sintassi letterale per ognuno sarebbe ripetitivo e inefficiente. Le Constructor Functions risolvono questo problema agendo come template.

*Proprietà di Istanza (tramite this)*

Le proprietà che variano da oggetto a oggetto (come il nome o l'età) vengono definite all'interno della funzione usando *this*. Ogni volta che usiamo new, queste istruzioni vengono eseguite, creando una "copia" dei dati per quella specifica istanza.

JavaScript

function User() {

// Proprietà specifiche per ogni istanza

this.name = "";

this.age = null;

}

let user1 = new User();

let user2 = new User();

// Personalizzazione delle istanze

user1.name = "Gianluca";

user2.name = "Simona";

console.log(user1.name); // "Gianluca"

console.log(user2.name); // "Simona"

*Proprietà Condivise (tramite prototype)*

Immaginiamo di voler aggiungere una proprietà type che sia *uguale per tutti* gli utenti (es. "Utente standard").

Approccio Inefficiente (dentro la funzione):

Se scrivessimo this.type = "Utente standard" dentro User, ogni volta che creiamo un nuovo utente (es. 1000 utenti), creeremmo 1000 copie identiche della stringa "Utente standard" in memoria. Questo è uno spreco di risorse.

Approccio Ottimizzato (Prototype):

Utilizziamo la proprietà speciale prototype. Le proprietà definite qui non vengono copiate in ogni oggetto, ma sono condivise. Gli oggetti vi accedono per "ereditarietà".

JavaScript

// Definizione della proprietà condivisa sul Prototipo

User.prototype.type = "Utente standard";

// Verifica

console.log(user1.type); // Output: "Utente standard"

console.log(user2.type); // Output: "Utente standard"

*Analisi Tecnica*

1. *Accesso:* Quando chiediamo user1.type, JavaScript cerca prima dentro l'oggetto user1.

2. *Lookup: Non trovandolo (perché non è stato definito con this.type), guarda nel suo prototipo*.

3. *Risultato:* Trova type in User.prototype e restituisce quel valore.

Questo meccanismo permette di risparmiare memoria mantenendo i dati comuni in un unico posto, mentre i dati specifici (name, age) risiedono nelle singole istanze.
