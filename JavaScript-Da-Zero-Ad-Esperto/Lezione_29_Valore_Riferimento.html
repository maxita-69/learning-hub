Passaggio Valore vs Riferimento

Lezione 29 - Corso JavaScript

*1\. Passaggio per Valore (Pass by Value)*

Nel passaggio per valore, ciò che viene trasferito è una *copia esatta del dato*2. Le due variabili coinvolte rimangono completamente indipendenti: una modifica alla variabile originale non ha alcun effetto sulla variabile copiata e viceversa.

*Tipi Coinvolti*

In JavaScript, questa modalità si applica a tutti i *Tipi Primitivi*:

  • Number
  • String
  • Boolean
  • Null
  • Undefined 3

*Esempio Pratico*

Analizziamo il comportamento di due variabili numeriche:

JavaScript

// Definiamo una variabile A con valore 10

let A = 10;

// Assegniamo A ad una nuova variabile B

let B = A;

// In questo momento, B riceve una COPIA del valore 10\.

// A e B sono indipendenti.

// Modifichiamo A

A = 8;

// Verifichiamo il valore di B

console.log(B); // Output: 10

Analisi:

L'istruzione let B = A non crea un legame tra le due variabili. Assegna a B solamente una copia del valore contenuto in A in quel preciso istante. Di conseguenza, la successiva modifica di A (che diventa 8\) non influisce minimamente su B, che mantiene il suo valore originale (10)4.

*2\. Passaggio per Riferimento (Pass by Reference)*

Il comportamento cambia radicalmente quando lavoriamo con strutture dati complesse. In JavaScript, *Oggetti, Array e Funzioni* vengono passati per riferimento5.

In questo contesto, una variabile non "contiene" fisicamente l'oggetto, ma contiene un *riferimento* (un puntatore o indirizzo) alla posizione in memoria dove quell'oggetto risiede6.

Quando assegniamo una variabile oggetto a un'altra:

1. Non viene creata una copia dell'oggetto.

2. Viene copiato solo il *riferimento* (l'indirizzo di memoria).

3. Entrambe le variabili puntano allo *stesso identico oggetto* in memoria7.

*Esempio di Mutazione Condivisa*

Questo implica che una modifica effettuata tramite una variabile si rifletterà immediatamente sull'altra, poiché entrambe "guardano" lo stesso dato.

JavaScript

// Definiamo un oggetto obj1 con una proprietà

let obj1 = { p1: 10 };

// Assegniamo obj1 a obj2

let obj2 = obj1;

// ORA: obj1 e obj2 puntano allo stesso indirizzo in memoria.

// Modifica tramite obj1

obj1.p1 = "20";

// Verifichiamo obj2

console.log(obj2.p1); // Output: "20"

// La modifica fatta su obj1 è visibile in obj2

// Modifica tramite obj2

obj2.p1 = "1.2";

// Verifichiamo obj1

console.log(obj1.p1); // Output: "1.2"

// La modifica fatta su obj2 si riflette su obj1

Come dimostrato dall'esempio, obj1 e obj2 sono due "telecomandi" diversi che controllano la stessa televisione. Se cambiamo canale con uno, il cambiamento è visibile anche per chi guarda il televisore (o usa l'altro telecomando)8.

*3\. Confronto e Uguaglianza tra Oggetti*

La natura del passaggio per riferimento ha implicazioni dirette sugli operatori di confronto (== o ===).

*Confronto tra Oggetti Distinti*

In JavaScript, due oggetti distinti non sono mai uguali, anche se il loro contenuto è identico.

JavaScript

// Confronto tra due oggetti vuoti distinti

let o1 = {};

let o2 = {};

console.log(o1 == o2); // false

console.log(o1 === o2); // false

*Perché?* JavaScript non controlla il contenuto (le proprietà interne), ma controlla se i riferimenti in memoria sono gli stessi. Poiché o1 e o2 sono stati creati separatamente, occupano posizioni di memoria diverse9.

*Confronto tra Riferimenti*

Il confronto restituisce true solo se le due variabili puntano allo stesso riferimento in memoria.

JavaScript

let obj1 = { valore: 10 };

let obj2 = obj1; // Copia del riferimento

// Confronto di uguaglianza (semplice o stretta)

console.log(obj1 == obj2); // true

console.log(obj1 === obj2); // true

In questo caso il risultato è true perché obj1 e obj2 sono alias per la stessa locazione di memoria10.
