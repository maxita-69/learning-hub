IIFE e Module Pattern

Lezione 36 - Corso JavaScript

*1\. Function Expression e Invocazione Immediata (IIFE)*

Una funzione in JavaScript può essere definita e invocata nello stesso istante. Questa tecnica è nota come *IIFE*.

*Analisi della Sintassi*

Per creare una IIFE, trasformiamo una Function Expression anonima in un'espressione valutabile racchiudendola tra parentesi tonde ( ... ), seguite da una coppia di parentesi () per l'invocazione.

*Esempio 1: Calcolo Immediato con Correzione Numerica*

Nel seguente esempio, creiamo una funzione che accetta due parametri, esegue un calcolo utilizzando un fattore locale e restituisce il risultato formattato.

*Ricostruzione Visiva:* A schermo viene mostrato l'uso dell'operatore unario + per convertire la stringa restituita da toFixed nuovamente in numero.

JavaScript

// La variabile 'calcola' riceverà il risultato dell'invocazione, non la funzione stessa

var calcola = (function(a, b) {

// Variabile locale, non accessibile dall'esterno

var fattore = 1.2;

// Esecuzione del calcolo

var ris = (a + b) * fattore;

// Risoluzione del problema dei decimali in JS (es. 7.1999...)

// 1\. .toFixed(2) -> restituisce la stringa "7.20"

// 2\. Operatore unario '+' -> converte la stringa in numero (7.2)

console.log(+ris.toFixed(2));

})(2, 4); // Invocazione immediata con parametri 2 e 4

*Output Console:* 7.2

*2\. Isolamento dello Scope (Privacy dei Dati)*

Il vantaggio principale delle IIFE è la creazione di uno *scope riservato. Le variabili definite al loro interno (come fattore nell'esempio sopra) nascono e muoiono con l'esecuzione della funzione e non sono visibili dal Contesto Globale* (Global Scope).

Se provassimo ad accedere a fattore dall'esterno, otterremmo un errore:

JavaScript

console.log(fattore);

// Uncaught ReferenceError: fattore is not defined

Questo meccanismo previene il *Global Namespace Pollution* (inquinamento dello spazio globale), evitando conflitti con altre variabili che potrebbero avere lo stesso nome.

*3\. Il Module Pattern*

Il *Module Pattern è un modello di progettazione che sfrutta le IIFE per implementare il concetto di incapsulamento*. Ci permette di:

1. Mantenere *private* le funzioni e le variabili interne.

2. Esporre *pubblicamente* solo un'interfaccia selezionata (API) tramite la restituzione di un oggetto.

*Esempio 2: Creazione di un Modulo "Chat" (File lib.js)*

Di seguito il codice completo ricostruito visualmente dall'editor del docente. Il modulo gestisce l'avvio di una chat simulata, distinguendo tra versione desktop e mobile.

JavaScript

// Assegniamo alla variabile 'chat' l'oggetto restituito dalla IIFE

var chat = (function() {

// --- MEMBRI PRIVATI (Nascosti) ---

// Queste funzioni non possono essere chiamate direttamente dall'esterno

function chat(tipo) {

if (tipo === 'mobile') {

startChatMobile();

} else {

startChat();

}

}

function startChat() {

console.log("Sto avviando la chat");

}

function startChatMobile() {

console.log("Sto avviando la chat mobile");

}

// --- API PUBBLICA (Esposta) ---

// Restituiamo un oggetto letterale che fa da "ponte" verso le funzioni private.

// La proprietà 'init' è l'unico punto di accesso per l'utente.

return {

init: chat

};

})();

*Verifica dell'Accesso*

Se proviamo a invocare le funzioni interne dal contesto globale:

  • chat.chat('mobile') $\\rightarrow$ *Errore* (funzione non esposta).
  • chat.startChat() $\\rightarrow$ *Errore*.

L'unico modo corretto per interagire con il modulo è usare la proprietà esposta init:

JavaScript

// Accesso corretto tramite l'API pubblica

chat.init('mobile');

// Output: "Sto avviando la chat mobile"

*Riepilogo Concettuale*

Con questo pattern, la variabile globale chat non contiene la logica interna, ma solo l'oggetto { init: chat }. Questo ci permette di *nascondere* la complessità (le funzioni startChat, startChatMobile) e proteggere il codice da interferenze esterne, esponendo solo ciò che è strettamente necessario.
