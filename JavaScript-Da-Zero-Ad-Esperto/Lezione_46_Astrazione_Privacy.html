# *L'Astrazione e la Gestione della Privacy nelle Constructor Functions*

In questo capitolo approfondiremo il concetto di *Astrazione*, uno dei quattro pilastri fondamentali della Programmazione Orientata agli Oggetti (OOP). Analizzeremo come implementare questo principio in JavaScript utilizzando le Constructor Functions, focalizzandoci sulla creazione di membri privati per nascondere i dettagli implementativi e proteggere la logica interna dell'oggetto.

*1\. Il Concetto di Astrazione*

L'astrazione è il principio secondo cui un sistema complesso dovrebbe nascondere i dettagli di funzionamento interno ed esporre all'utente solo l'interfaccia necessaria per l'interazione.

*L'Analogia del Lettore DVD*

Per comprendere il concetto, la lezione utilizza l'esempio di un lettore DVD. Sebbene la progettazione interna del dispositivo sia estremamente complessa (circuiti, laser, motori), l'interfaccia esposta all'utente è semplice: pochi pulsanti intuitivi (Play, Stop, Eject).

L'astrazione consiste proprio in questo: nascondere la complessità ed esporre solo l'essenziale.

In JavaScript, l'obiettivo è applicare la stessa logica agli oggetti: permettere l'utilizzo delle funzionalità (metodi pubblici) impedendo l'accesso diretto o la modifica delle logiche interne (variabili o funzioni private).

*2\. Implementazione della Privacy nei Costruttori*

JavaScript (nelle versioni storiche e nell'uso delle funzioni costruttore) non possiede modificatori di visibilità nativi come public o private tipici di altri linguaggi (Java o C\#). Tuttavia, possiamo simulare la privacy sfruttando lo *scope delle funzioni*.

*Il Problema: Tutto è Pubblico per Default*

Quando definiamo proprietà e metodi usando la keyword this all'interno di una Constructor Function, questi diventano automaticamente pubblici e accessibili dall'esterno.

![][image1]

*La Soluzione: Utilizzo di Variabili Locali (let)*

Per rendere "privati" dei dati o delle funzioni, dobbiamo smettere di allegarli all'oggetto tramite this. Invece, li dichiariamo come *variabili locali* all'interno della funzione costruttore usando let (o const/var).

In questo modo, tali variabili saranno visibili solo all'interno della funzione User (grazie alla chiusura lessicale), ma non saranno accessibili tramite l'istanza dell'oggetto.

*Nota Tecnica sulla Sintassi*

Nel passaggio dal codice pubblico a quello privato, è fondamentale notare una correzione sintattica descritta nella lezione:

  • Quando calcolaPesoIdeale era pubblico (this.calcolaPesoIdeale), doveva essere invocato con this.calcolaPesoIdeale().
  • Una volta trasformato in variabile locale (let calcolaPesoIdeale), *il riferimento this va rimosso* durante l'invocazione. La funzione esiste nello scope locale, non sull'oggetto.

*Approfondimenti e Note Integrative*

*Il Meccanismo della Closure*

La tecnica descritta in questa lezione funziona grazie al concetto di *Closure* (chiusura). Quando il costruttore User termina l'esecuzione, le variabili locali (calcolaPesoIdeale, privata, altezza) dovrebbero teoricamente essere distrutte. Tuttavia, poiché il metodo pubblico this.pesoIdeale mantiene un riferimento a esse, queste variabili vengono "mantenute in vita" in memoria, accessibili solo da quel metodo specifico.

*Svantaggi Prestazionali*

Definire metodi (anche pubblici) all'interno del costruttore, come fatto per this.pesoIdeale, comporta che la funzione venga *ricreata in memoria per ogni singola istanza* (user1, user2, user3 avranno copie separate della funzione). Sebbene necessario per accedere alle variabili private (che sono uniche per istanza), questo approccio consuma più memoria rispetto all'uso del prototype.

*Evoluzione Moderna: Classi e Campi Privati (\#)*

La lezione menziona che JavaScript avrebbe integrato "a breve" meccanismi nativi. Nelle versioni moderne di ECMAScript (ES2022+), è stata introdotta la sintassi delle classi con i *campi privati*, identificati dal prefisso \#.

JavaScript

class User {

\#altezza; // Campo privato nativo

constructor(name, altezza) {

this.name = name;

this.\#altezza = altezza;

}

\#calcolaPeso() { // Metodo privato nativo

return (this.\#altezza - 100) * 0.95;

}

getPeso() {

return this.\#calcolaPeso();

}

}
