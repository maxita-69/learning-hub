Funzioni: Definizione e Natura

Lezione 32 - Corso JavaScript

*1\. Definizione e Natura delle Funzioni*

Una funzione è essenzialmente un *sottoprogramma* progettato per svolgere determinati compiti in un ambiente circoscritto.

Le caratteristiche fondamentali sono:

  • *Definizione e Invocazione:* Le funzioni vengono prima definite e successivamente eseguite tramite invocazione.
  • *Input e Output: Possono ricevere dati in ingresso (parametri), ma restituiscono sempre* un dato in uscita (output). Se non viene specificato un return, la funzione restituisce implicitamente undefined.
  • *Natura ad Oggetti: In JavaScript, è fondamentale comprendere che le funzioni sono Oggetti*.

*2\. Funzioni come "First-Class Citizens"*

In JavaScript, le funzioni sono definite *Oggetti di Prima Classe (o Cittadini di Prima Classe*).

*Cosa significa?*

Nel mondo della programmazione, un'entità è considerata di "prima classe" quando supporta tutte le modalità operative delle altre entità standard (come stringhe, numeri, booleani).

In termini pratici, tutto ciò che puoi fare con un numero o una stringa, puoi farlo con una funzione:

1. Assegnarla a una variabile.

2. Passarla come parametro ad altre funzioni.

3. Restituirla come risultato di un'altra funzione.

*3\. Function Expressions (Espressioni di Funzione)*

Sfruttando il concetto di First-Class Citizen, possiamo utilizzare una sintassi alternativa alla classica dichiarazione: la Function Expression.

Si tratta di assegnare una funzione (spesso anonima, cioè senza nome) direttamente ad una variabile.

*Codice Esemplicativo*

JavaScript

// Definiamo una variabile 'fn' e le assegniamo una funzione anonima

let fn = function() {

console.log("Esecuzione della funzione anonima");

};

// NOTA: Qui è necessario il punto e virgola (;) finale,

// perché stiamo tecnicamente facendo un assegnamento di valore a una variabile.

// 'fn' ora conserva il riferimento alla funzione.

// Per eseguirla, la invochiamo usando le parentesi tonde:

fn();
