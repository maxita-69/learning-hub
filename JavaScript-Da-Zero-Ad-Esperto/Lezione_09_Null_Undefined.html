*Lezione: null e undefined - Assenza di Valore*

*1\. Introduzione a null e undefined*

Sono tipi primitivi che indicano assenza di valore e che generano un po' di confusione. Non letteralmente:

  • null vuol dire "nullo"
  • undefined vuol dire "non definito"

*2\. Differenza Fondamentale*

null è un valore che deve essere assegnato esplicitamente:

let totale = null;

console.log(totale); // null

undefined, invece, significa che una variabile è stata dichiarata, ma non le è stato assegnato alcun valore:

javascript

let a; // dichiariamo la variabile ma non assegniamo alcun valore

console.log(a); // undefined

*3\. Altri Casi che Restituiscono undefined*

Accesso a proprietà inesistente di un oggetto:

let oggetto = {};

console.log(oggetto.totale); // undefined

Funzione senza return:

javascript

function test() {  
    // nessun return  
}

console.log(test()); // undefined

*4\. Caratteristiche in Comune*

Le caratteristiche in comune che hanno null e undefined sono:

1. Entrambi sono valori primitivi

2. Entrambi sono considerati falsy

Verifica nella console:

\!\!undefined; // false

\!\!null;      // false

Se trasformiamo undefined nel booleano avremo false, e se facciamo la stessa cosa per null, allo stesso modo avremo false.

*5\. Conclusione*

Anche se entrambi rappresentano assenza di valore, è importante distinguere:

  • null: valore intenzionale di "nessun valore"
  • undefined: valore non ancora assegnato o non definito
