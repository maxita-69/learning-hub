*Lezione: BigInt - Superare i Limiti dei Numeri*

*1\. Introduzione a BigInt*

In ECMAScript 2020 (ES11) è stato introdotto il tipo primitivo BigInt per superare i limiti del tipo number.

Come possiamo leggere su MDN (Mozilla Developer Network), che è la piattaforma Mozilla di documentazione e apprendimento delle tecnologie per il web, abbiamo ad oggi (sto registrando a novembre 2021\) sette tipi primitivi, tra cui BigInt.

I valori primitivi sono:

  • Stringhe
  • Numeri
  • BigInt
  • Valori booleani
  • Undefined
  • Symbol
  • Null

*2\. Cos'è BigInt?*

Clicco su BigInt e leggiamo che BigInt è un tipo di dato numerico che può rappresentare interi in un formato di precisione arbitrario.

Che cosa vuol dire? Possiamo cliccare sul link e leggere le informazioni su Wikipedia, ma in poche parole vuol dire che possiamo utilizzare qualsiasi valore numerico, anche molto, molto grande, e quindi possiamo effettuare anche operazioni aritmetiche tra valori molto grandi senza avere alcun problema, come invece abbiamo per il tipo number.

*3\. I Limiti del Tipo Number*

Infatti se apro la console con F12, per quanto riguarda il tipo number abbiamo un range entro il quale possiamo effettuare operazioni aritmetiche senza alcun problema.

Abbiamo:

  • Limite inferiore: Number.MIN_SAFE_INTEGER
  • Limite superiore: Number.MAX_SAFE_INTEGER

Dimostrazione pratica:

let num1 = Number.MAX_SAFE_INTEGER;  
console.log(num1); // 9007199254740991

console.log(num1 + 1); // 9007199254740992 - Corretto

console.log(num1 + 2); // 9007199254740992 - ERRATO\! Non è corretto

Alla fine dovremmo avere 9007199254740993, ma il risultato non è corretto. Questi tipi di problemi non li abbiamo con BigInt.

*4\. Come Definire un BigInt*

Cancello e pulisco la console. Come facciamo a definire un tipo BigInt? Possiamo farlo in due modi:

Primo modo - Notazione letterale (aggiungendo n alla fine del numero):

let num2 = 10n;  
console.log(num2); // 10n

// Verifichiamo il tipo

console.log(typeof num2); // bigint

Secondo modo - Utilizzando la funzione BigInt():

javascript

let num4 = BigInt(20);  
console.log(num4); // 20n

let num5 = BigInt(Number.MAX_SAFE_INTEGER);

console.log(num5); // 9007199254740991n

*5\. Operazioni con BigInt*

Ora proviamo a fare operazioni:

console.log(num5 + 1); // ERRORE\! Non possiamo sommare BigInt a Number

console.log(num5 + 1n); // 9007199254740992n - CORRETTO\!

console.log(num5 + 2n); // 9007199254740993n - CORRETTO\!

Anche se siamo oltre al valore di Number.MAX_SAFE_INTEGER, non abbiamo problemi ad effettuare operazioni aritmetiche.

*6\. Tutte le Operazioni Aritmetiche con BigInt*

Che cosa possiamo fare? Quali operazioni aritmetiche possiamo effettuare con valori BigInt? Tutte quelle che conosciamo.

Addizioni:

console.log(2n + 4n); // 6n

Attenzione: Non possiamo sommare BigInt a Number:

javascript

console.log(2n + 1); // ERRORE: Non possiamo convertire BigInt in Number

Moltiplicazioni:

javascript

console.log(2n  3n); // 6n*

Sottrazioni:

javascript

console.log(4n - 10n); // -6n

Operatori di incremento e decremento:

javascript

let num6 = 30n;  
console.log(num6--); // 30n (post-decremento)  
console.log(num6);   // 29n

let num7 = 10n;  
console.log(++num7); // 11n (pre-incremento)

console.log(++num7); // 12n

Divisioni:

javascript

console.log(10n / 5n); // 2n

console.log(10n / 6n); // 1n - Perdiamo la parte decimale

Perché perdiamo la parte decimale? Quando parliamo di BigInt parliamo di valori interi. Infatti la parola stessa dice BigInt per "integer" (intero). Quindi possiamo effettuare divisioni, ma la parte decimale verrà troncata.

Elevazione a potenza:

javascript

console.log(2n * 4n); // 16n*

Operatore modulo:

javascript

console.log(10n % 3n); // 1n

*7\. Conclusione*

Perfetto\! Abbiamo visto come BigInt risolve i problemi di precisione con numeri molto grandi e tutte le operazioni che possiamo effettuare con questo tipo di dato.
