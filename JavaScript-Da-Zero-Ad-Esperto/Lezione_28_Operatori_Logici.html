Operatori Logici

Lezione 28 - Corso JavaScript

Sebbene vengano comunemente associati alla logica booleana pura (vero/falso), in JavaScript il loro comportamento è più sfumato: essi effettuano internamente una conversione (coercizione) dei valori per valutarli, ma alla fine restituiscono il valore originale di una delle espressioni coinvolte.

*1\. L'Operatore AND (&&)*

L'operatore logico AND (&&) si applica su due espressioni. La sua logica di valutazione segue questa regola fondamentale:

*Regola dell'AND: Se la prima espressione può essere convertita a true, l'operatore restituisce la seconda espressione. Altrimenti, restituisce la prima espressione*.

*Esempi di Valutazione*

Analizziamo il comportamento con valori misti (numeri, stringhe):

JavaScript

// Esempio 1: Primo valore Truthy

2 && 0;

// Risultato: 0

// Spiegazione: 2 è truthy. Quindi viene restituita la seconda espressione (0).

// Esempio 2: Primo valore Truthy

2 && 1;

// Risultato: 1

// Spiegazione: 2 è truthy. Restituisce la seconda (1).

// Esempio 3: Primo valore Truthy con stringa

2 && "CIAO";

// Risultato: "CIAO"

// Spiegazione: 2 è truthy. Restituisce "CIAO".

Ovviamente, se lavoriamo con booleani puri, il comportamento rispecchia la classica tabella di verità:

JavaScript

true && false; // false

false && true; // false

true && true; // true

*2\. L'Operatore OR (||)*

*Regola dell'OR: Se la prima espressione può essere valutata a true, l'operatore restituisce immediatamente la prima espressione (cortocircuito). Altrimenti, restituisce la seconda espressione*.

*Esempi di Valutazione*

JavaScript

// Esempio 1: Primo valore Truthy

// Risultato: 2

// Spiegazione: 2 è truthy. L'operatore si ferma e restituisce 2\.

// Esempio 2: Primo valore Falsy

// Risultato: 2

// Spiegazione: 0 è falsy. L'operatore passa alla seconda espressione e restituisce 2\.

// Esempio 3: Valori speciali

// Risultato: undefined

// Spiegazione: 0 è falsy. Viene restituita la seconda espressione, anche se è undefined.

*3\. L'Operatore NOT (\!)*

A differenza di AND e OR, il NOT (\!) è un *operatore unario*, ovvero agisce su una singola espressione. La sua funzione è invertire il valore di verità.

*Regola del NOT:* Se l'espressione può essere convertita a false, restituisce true. Se può essere convertita a true, restituisce false.

JavaScript

\!0; // true

// Spiegazione: 0 è falsy -> il NOT lo inverte in true.

\!10; // false

// Spiegazione: 10 è truthy -> il NOT lo inverte in false.

\!true; // false

\!false; // true

*4\. Applicazione Pratica: Combinare Condizioni*

Gli operatori logici sono essenziali per combinare condizioni multiple all'interno delle strutture di controllo, come l'istruzione if.

*Esempio Base: Verifica del Voto*

Immaginiamo di voler verificare se un voto v è compreso tra 18 e 22 per decidere se fare una passeggiata.

JavaScript

let v = 19;

if (v >= 18 && v < 22) {

console.log("Passeggiata");

}

*Analisi passo-passo:*

1. v >= 18 (19 è maggiore di 18\) → *TRUE*

2. v < 22 (19 è minore di 22\) → *TRUE*

3. Risultato finale: TRUE && TRUE → *TRUE*.

4. L'istruzione viene eseguita e stampa "Passeggiata".

*Esempio Avanzato: Raggruppamento con Parentesi*

Aggiungiamo complessità: vogliamo fare una passeggiata se il voto è buono *E se la giornata è di sole OPPURE* nuvolosa (ma non piovosa).

Quando si combinano AND e OR, è buona norma (e spesso necessario per la precedenza degli operatori) utilizzare le *parentesi tonde* per raggruppare le condizioni logiche.

JavaScript

let v = 19;

let g = "pioggia"; // Meteo attuale

// Condizione complessa con raggruppamento

console.log("Passeggiata");

}
