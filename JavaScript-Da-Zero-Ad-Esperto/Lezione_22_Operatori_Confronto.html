Operatori di Confronto

Lezione 22 - Corso JavaScript

*1\. Classificazione degli Operatori*

Prima di entrare nel merito dei confronti, è utile distinguere gli operatori in base al numero di operandi coinvolti:

  • *Operatori Unari:* Si applicano a un singolo elemento.
  • Esempio: L'operatore di incremento (++) che aumenta di uno il valore di una variabile, o l'operatore di negazione logica doppia (\!\!) per convertire un valore in booleano.
  • *Operatori Binari:* Operano su un'espressione composta da due elementi. Il risultato di queste operazioni è sempre un valore booleano: true (vero) o false (falso).

*2\. Operatori di Confronto Relazionale*

Gli operatori binari di confronto (<, <=, >, >=) verificano la relazione tra due valori.

*Confronto Numerico*

È il confronto più intuitivo e segue le regole matematiche standard.

JavaScript

console.log(3 > 3); // false

console.log(3 >= 3); // true

console.log(3 < 4); // true

*Confronto tra Stringhe*

Quando gli operandi sono stringhe, JavaScript non confronta il significato delle parole, ma il valore del codice *ASCII dei caratteri. Questo significa che il confronto è case-sensitive* (sensibile alle maiuscole/minuscole).

JavaScript

// La 'a' minuscola ha un codice ASCII maggiore della 'A' maiuscola

console.log('a' > 'A'); // true

*3\. L'Uguaglianza Semplice (==) e la Coercizione*

L'operatore di uguaglianza semplice (==) verifica se due valori sono uguali. Tuttavia, se i tipi sono diversi, JavaScript tenta di effettuare una *coercizione implicita* (trasformazione automatica) per renderli confrontabili.

Questo comportamento può portare a risultati apparentemente strani, ma logici se si conoscono le regole di conversione:

JavaScript

// Conversione Stringa -> Numero

console.log("3" == 3); // true

// Coercizione con caratteri speciali (es. Newline)

// JavaScript ignora gli spazi bianchi e i ritorni a capo (\\n) nella conversione numerica

console.log("3\\n" == 3); // true

// Gestione dello Zero

console.log(-0 == 0); // true

// Null e Undefined

console.log(null == undefined); // true

// NaN (Not a Number) è l'unico valore non uguale a se stesso

console.log(NaN == NaN); // false

*4\. L'Uguaglianza Stretta (===)*

Per evitare le ambiguità della coercizione implicita, esiste l'operatore di uguaglianza stretta (===).

Questo operatore non effettua alcuna conversione: restituisce true solo se sia il valore che il tipo sono identici.

JavaScript

// Confronto tra tipi diversi

console.log("0" === 0); // false (Stringa vs Numero)

// Confronto tra valori identici dello stesso tipo

console.log(0 === 0); // true

console.log(0 === 1); // false

*Capitolo: Gli Operatori Binari di Confronto ed Uguaglianza in Profondità*

In JavaScript, la capacità di confrontare valori è fondamentale per la logica di controllo del flusso. Questo capitolo esplora in modo esaustivo come il linguaggio gestisce queste operazioni, analizzando gli operatori di confronto, la dinamica della coercizione implicita e la distinzione cruciale tra uguaglianza semplice e stretta.*1\. Classificazione degli Operatori: Unari vs. Binari*

Per comprendere appieno le operazioni di confronto, è essenziale distinguere gli operatori in base al numero di operandi su cui agiscono:

  • *Operatori Unari:* Richiedono un singolo operando. Modificano o valutano l'elemento isolato.
  • Esempi:
  • L'operatore di incremento (`++`) o decremento (`--`). Esempio: `x++` (agisce solo su `x`).
  • L'operatore di negazione logica singola (`!`) per invertire il valore booleano di un'espressione, o la doppia negazione (`!!`) usata come type casting rapido per convertire qualsiasi valore nel suo equivalente booleano (`true` o `false`).
